# Wire ‚Äî Platform Credentials Setup Guide

This guide walks you through getting every credential needed to run Wire locally. Fill values into `.env.local` as you go.

---

## 1. Clerk (Authentication)

> Already configured ‚Äî skip unless starting fresh.

1. Go to [clerk.com](https://clerk.com) ‚Üí Sign in
2. Create a new application (or select existing)
3. Dashboard ‚Üí **API Keys**:
   - Copy **Publishable Key** ‚Üí `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`
   - Copy **Secret Key** ‚Üí `CLERK_SECRET_KEY`
4. Dashboard ‚Üí **Webhooks** ‚Üí Create endpoint:
   - URL: `https://YOUR_DOMAIN/api/webhooks/clerk`
   - Events: `user.created`, `user.updated`, `user.deleted`
   - Copy **Signing Secret** ‚Üí `CLERK_WEBHOOK_SECRET`
5. Dashboard ‚Üí **JWT Templates** ‚Üí Copy **Issuer domain** ‚Üí `CLERK_JWT_ISSUER_DOMAIN`

Push to Convex:
```bash
npx convex env set CLERK_JWT_ISSUER_DOMAIN "https://your-issuer.clerk.accounts.dev"
```

## 2. Google / Gmail

> Already configured ‚Äî skip unless starting fresh.

### Step A: Create OAuth Credentials

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Select your project (or create one)
3. Navigate ‚Üí **APIs & Services** ‚Üí **Enabled APIs & Services**
4. Click **+ ENABLE APIS AND SERVICES** ‚Üí Search and enable:
   - **Gmail API**
   - **Google People API** (if needed)
5. Navigate ‚Üí **APIs & Services** ‚Üí **Credentials**
6. Click **+ CREATE CREDENTIALS** ‚Üí **OAuth client ID**
7. If prompted, configure the **OAuth consent screen** first:
   - User type: **External** (for testing) or **Internal** (if in a Workspace)
   - App name: `Wire`
   - Scopes: `https://www.googleapis.com/auth/gmail.readonly`, `https://www.googleapis.com/auth/gmail.send`, `https://www.googleapis.com/auth/gmail.modify`
8. Back to Credentials ‚Üí **OAuth client ID**:
   - Application type: **Web application**
   - Authorized redirect URIs: `http://localhost:3000/api/auth/google/callback`
9. Copy values:
   - **Client ID** ‚Üí `GOOGLE_CLIENT_ID`
   - **Client Secret** ‚Üí `GOOGLE_CLIENT_SECRET`

### Step B: Gmail Pub/Sub (real-time push notifications)

1. Navigate ‚Üí [Pub/Sub Console](https://console.cloud.google.com/cloudpubsub/topic/list)
2. Click **CREATE TOPIC**
   - Topic ID: `gmail-watch`
3. Click on the new topic ‚Üí **Permissions** tab ‚Üí **Add Principal**
   - Principal: `gmail-api-push@system.gserviceaccount.com`
   - Role: **Pub/Sub Publisher**
4. Create a **Subscription** on the topic:
   - Delivery type: Push
   - Endpoint URL: `https://YOUR_DOMAIN/api/webhooks/gmail`
5. Set env vars:
   - `GMAIL_PUBSUB_TOPIC=projects/YOUR_PROJECT_ID/topics/gmail-watch`
   - `GOOGLE_CLOUD_PROJECT_ID=YOUR_PROJECT_ID` (visible in GCP dashboard URL)

Push to Convex:
```bash
npx convex env set GOOGLE_CLIENT_ID "your-client-id"
npx convex env set GOOGLE_CLIENT_SECRET "your-client-secret"
npx convex env set GMAIL_PUBSUB_TOPIC "projects/your-project/topics/gmail-watch"
```

## 3. Slack

> Already configured ‚Äî skip unless starting fresh.

1. Go to [api.slack.com/apps](https://api.slack.com/apps) ‚Üí **Create New App** ‚Üí **From scratch**
2. Name it `Wire`, select your workspace
3. **Basic Information** page ‚Äî scroll to **App Credentials**:
   - Copy **Client ID** ‚Üí `SLACK_CLIENT_ID`
   - Copy **Client Secret** ‚Üí `SLACK_CLIENT_SECRET`
   - Copy **Signing Secret** ‚Üí `SLACK_SIGNING_SECRET`
4. **OAuth & Permissions** ‚Üí Add these **Bot Token Scopes**:
   - `users:read` ‚Äî list workspace members
   - `im:read` ‚Äî read DM metadata
   - `im:history` ‚Äî read DM messages
   - `chat:write` ‚Äî send messages
   - `users.profile:read` ‚Äî read profile info
5. **OAuth & Permissions** ‚Üí **Redirect URLs** ‚Üí Add:
   - `http://localhost:3000/api/auth/slack/callback`
6. **Event Subscriptions** ‚Üí Enable Events ‚Üí Request URL:
   - `https://YOUR_DOMAIN/api/webhooks/slack`
   - Subscribe to bot events: `message.im`, `message.channels`
7. **Install to Workspace** (if not already)

Push to Convex:
```bash
npx convex env set SLACK_CLIENT_ID "your-client-id"
npx convex env set SLACK_CLIENT_SECRET "your-client-secret"
```

## 4. WhatsApp (via Twilio) ‚Äî NEW ‚≠ê

### Step A: Create Twilio Account

1. Go to [twilio.com](https://www.twilio.com/) ‚Üí Sign up for a free account
2. Verify your phone number and email
3. On the **Console Dashboard** (first page after login), you'll see:
   - **Account SID** (starts with `AC`) ‚Üí `TWILIO_ACCOUNT_SID`
   - **Auth Token** (click "Show") ‚Üí `TWILIO_AUTH_TOKEN`

### Step B: Set Up WhatsApp Sandbox (for testing)

1. In Twilio Console ‚Üí left sidebar ‚Üí **Messaging** ‚Üí **Try it out** ‚Üí **Send a WhatsApp message**
2. Follow the instructions to join the sandbox:
   - Send the displayed code (e.g., `join powder-cake`) from your WhatsApp to the sandbox number
3. The sandbox number will look like `+14155238886`
4. Set `TWILIO_WHATSAPP_FROM=whatsapp:+14155238886`

### Step C: Configure Webhook

1. In Twilio Console ‚Üí **Phone Numbers** ‚Üí **Active Numbers** ‚Üí click the WhatsApp sandbox number
2. Scroll to **Messaging Configuration**
3. Under **"When a message comes in"**:
   - Method: POST
   - URL: `https://YOUR_DOMAIN/api/webhooks/whatsapp`
4. For local dev, use ngrok: `ngrok http 3000` and use the https URL

### Step D: For Production (optional)

1. Purchase a Twilio phone number with WhatsApp capability
2. Apply for a WhatsApp Business API profile in Twilio Console
3. Update `TWILIO_WHATSAPP_FROM` with your production number

Set in `.env.local`:
```
TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_AUTH_TOKEN=your_auth_token_here
TWILIO_WHATSAPP_FROM=whatsapp:+14155238886
```

Push to Convex:
```bash
npx convex env set TWILIO_ACCOUNT_SID "ACxxxxxxxx"
npx convex env set TWILIO_AUTH_TOKEN "your_auth_token"
npx convex env set TWILIO_WHATSAPP_FROM "whatsapp:+14155238886"
```

## 5. Discord ‚Äî NEW ‚≠ê

### Step A: Create a Discord Application

1. Go to [discord.com/developers/applications](https://discord.com/developers/applications)
2. Click **New Application** ‚Üí Name it `Wire` ‚Üí Create
3. You're now on the **General Information** page:
   - Copy **APPLICATION ID** ‚Üí `NEXT_PUBLIC_DISCORD_CLIENT_ID`
   - Copy **PUBLIC KEY** ‚Üí `DISCORD_PUBLIC_KEY`

### Step B: Get OAuth2 Credentials

1. Left sidebar ‚Üí **OAuth2**
2. Copy **CLIENT SECRET** (click "Reset Secret" if needed) ‚Üí `DISCORD_CLIENT_SECRET`
3. Under **Redirects** ‚Üí Add Redirect:
   - `http://localhost:3000/api/auth/discord/callback`
4. Save Changes

### Step C: Create a Bot

1. Left sidebar ‚Üí **Bot**
2. Click **Reset Token** ‚Üí Copy the token ‚Üí `DISCORD_BOT_TOKEN`
   > ‚öÝÔ∏è This token is shown only once! Save it immediately.
3. Scroll down to **Privileged Gateway Intents**:
   - Enable **MESSAGE CONTENT INTENT** (required to read DM text)
   - Enable **SERVER MEMBERS INTENT** (optional)
4. Save Changes

### Step D: Invite the Bot to Your Server

1. Left sidebar ‚Üí **OAuth2** ‚Üí **URL Generator**
2. Select scopes: `bot`, `applications.commands`
3. Select bot permissions: `Read Messages/View Channels`, `Send Messages`, `Read Message History`
4. Copy the generated URL ‚Üí open in browser ‚Üí select your test server ‚Üí Authorize

### Step E: Configure Interactions Endpoint (optional, for webhooks)

1. Left sidebar ‚Üí **General Information**
2. **Interactions Endpoint URL**: `https://YOUR_DOMAIN/api/webhooks/discord`
3. Discord will send a verification ping ‚Äî your webhook handler already responds to `type: 1` pings

Set in `.env.local`:
```
NEXT_PUBLIC_DISCORD_CLIENT_ID=123456789012345678
DISCORD_CLIENT_SECRET=abcdefghijklmnop
DISCORD_PUBLIC_KEY=abcdef123456...
DISCORD_BOT_TOKEN=MTIzNDU2Nzg5MDEyMzQ1Njc4.XXXXXX.XXXXXXXX
```

Push to Convex:
```bash
npx convex env set DISCORD_BOT_TOKEN "your_bot_token"
```
